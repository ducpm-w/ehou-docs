Đề xuất năm 1945, mọi máy tính từ đó đến nay đều dựa trên thiết kế này. Gồm 3 thành phần chính kết nối qua Bus hệ thống.

#Nguyên lýNội dung

2 Thực thi tuần tự CPU đọc từng lệnh một theo thứ tự từ bộ nhớ, giải mã và thực thi — lặp đi lặp lại liên tục.

Hệ nhị phân & Bit

Máy tính chỉ có 2 trạng thái điện: có điện (1) và không có điện (0). Mọi thứ — số, chữ, hình ảnh, video — đều phải quy về chuỗi 0 và 1.

Đơn vịKích thướcMô tả
Bit1 bitĐơn vị nhỏ nhất. Chỉ có 2 giá trị: 0 hoặc 1. Ví dụ: 1 bóng đèn — tắt (0) hoặc sáng (1).
Byte8 bitĐơn vị cơ bản thường dùng. 1 byte = 8 bit.
Word16 / 32 / 64 bitSố bit CPU xử lý trong một lần. Tùy kiến trúc CPU.
Ví dụ: số 5 viết bằng 8 bit 0 0 0 0 0 1 0 1
Giá trị = 4 + 1 = 5 ✓ (bit màu xanh = 1, bit xám = 0)

Chuyển đổi cơ số

Thập phân → Nhị phân (phần nguyên)

Quy tắc: Chia liên tiếp cho 2, ghi phần dư. Đọc phần dư từ dưới lên.

Ví dụ: 19₁₀ → nhị phân 19 ÷ 2 = 9 dư 1 ← đọc sau 9 ÷ 2 = 4 dư 1 4 ÷ 2 = 2 dư 0 2 ÷ 2 = 1 dư 0 1 ÷ 2 = 0 dư 1 ← đọc trước → 19₁₀ = 10011₂

Kiểm tra: 10011₂ = 1×16 + 0×8 + 0×4 + 1×2 + 1×1 = 16 + 2 + 1 = 19 ✓

Thập phân → Nhị phân (phần thập phân)

Quy tắc: Nhân liên tiếp với 2, lấy phần nguyên. Đọc từ trên xuống.

Ví dụ: 0.625₁₀ → nhị phân 0.625 × 2 = 1.25 → lấy 1, dư 0.25 0.25 × 2 = 0.50 → lấy 0, dư 0.50 0.50 × 2 = 1.00 → lấy 1, dư 0.00 ← DỪNG → 0.625₁₀ = 0.101₂
Khi nào dừng?Giải thích
Phần dư = 0Kết quả chính xác, dừng ngay.
Phần dư lặp lạiSố vô hạn tuần hoàn — không thể biểu diễn chính xác.
Đủ n bit yêu cầuLàm tròn và dừng theo yêu cầu bài.
Lưu ý quan trọng Số như 0.1, 0.107, 0.3 không có biểu diễn nhị phân hữu hạn. Đó là lý do 0.1 + 0.2 ≠ 0.3 trong máy tính!

Số âm & Bù 2 (Two's Complement)

ALU bên trong CPU chỉ có mạch cộng. Khi tính 5 − 3, máy tính phải thực hiện 5 + (−3). Vì vậy cần một cách biểu diễn số âm bằng 0 và 1.

Ý tưởng bù 2 Trong hệ 4-bit (0–15): −3 và +13 có tác dụng giống hệt nhau khi cộng — giống kim đồng hồ quay thuận 13 bước hay ngược 3 bước đều đến cùng một chỗ.

Cách tính bù 2 (ví dụ: −5 trong 8 bit)

BướcThao tácKết quả
Bước 0Viết +5 ra nhị phân (8 bit)0 0 0 0 0 1 0 1
Bước 1Đảo tất cả bit (bù 1)1 1 1 1 1 0 1 0
Bước 2Cộng thêm 11 1 1 1 1 0 1 1 ← đây là −5
Kiểm tra: (+5) + (−5) = ? 0 0 0 0 0 1 0 1 (+5) + 1 1 1 1 1 0 1 1 (−5) ───────────────── 1 0 0 0 0 0 0 0 0 ↑ bit thứ 9 rớt ra → còn lại: 00000000 = 0 ✓
Quy tắc đọc bit dấu Bit đầu tiên (bit cao nhất) = 0 → số dương.
Bit đầu tiên = 1 → số âm.

Số thực — Chuẩn IEEE 754 (32 bit)

Chuẩn quốc tế để biểu diễn số dấu phẩy động trong máy tính. Số 32-bit chia thành 3 trường:

TrườngSố bitVị tríÝ nghĩa
S (Sign)1 bitBit 31Dấu: 0 = dương, 1 = âm
E (Exponent)8 bitBit 30–23Số mũ (có độ lệch bias = 127)
F (Fraction)23 bitBit 22–0Định trị (phần thập phân sau dấu phẩy)
Công thức V = (−1)S × 2(E−127) × 1.F
Giá trị đặc biệtĐiều kiện
±0E = 0, F = 0
±Vô cùng (∞)E = 255, F = 0
NaN (Not a Number)E = 255, F ≠ 0

CPU — 8 thành phần cốt lõi

CPU gồm nhiều thanh ghi (register) và hai đơn vị chức năng chính. Nhớ theo thứ tự xử lý:

Thứ tự hoạt động PC chỉ đường → IR nhận lệnh → CU giải mã → MAR/MBR lấy dữ liệu → ALU tính → ACC giữ kết quả
PC
Program Counter — trỏ đến địa chỉ lệnh tiếp theo cần thực thi.
IR
Instruction Register — giữ lệnh đang được thực thi.
MAR
Memory Address Register — giữ địa chỉ ô nhớ cần đọc hoặc ghi.
MBR
Memory Buffer Register — đệm dữ liệu khi đọc/ghi từ bộ nhớ.
ACC
Accumulator — lưu kết quả tính toán của ALU.
PSW
Program Status Word — cờ trạng thái: Z (zero), C (carry), S (sign).
CU
Control Unit — giải mã lệnh, tạo tín hiệu điều khiển cho toàn bộ CPU.
ALU
Arithmetic Logic Unit — thực hiện phép toán số học và logic.

Bus hệ thống — 3 loại

Bus là "đường cao tốc" nối CPU, bộ nhớ và thiết bị vào-ra. Có 3 loại bus với chức năng khác nhau:

Địa chỉ
Bus địa chỉ — truyền địa chỉ của ô nhớ hoặc thiết bị. Một chiều: CPU → Bộ nhớ.
Độ rộng bus xác định dung lượng nhớ tối đa (32-bit bus → tối đa 4 GB RAM).
Dữ liệu
Bus dữ liệu — truyền dữ liệu và lệnh. Hai chiều: CPU ↔ Bộ nhớ ↔ Thiết bị.
Độ rộng bus = kích thước word của CPU.
Điều khiển
Bus điều khiển — truyền tín hiệu điều khiển: đọc (RD), ghi (WR), xung nhịp (CLK), ngắt (INT)...
Hai chiều.

Vi thao tác (Micro-operation)

Mỗi bước trong chu kỳ lệnh được chia nhỏ thành vi thao tác — thao tác cơ bản nhất CPU có thể làm trong 1 xung nhịp đồng hồ.

Chu kỳ nhập lệnh (luôn xảy ra)

t1: MAR ← PC ← PC chỉ địa chỉ lệnh cần lấy t2: MBR ← M[MAR] ← đọc lệnh từ bộ nhớ vào MBR IR ← MBR[OP] ← lưu mã lệnh (opcode) vào IR PC ← PC + 1 ← PC chỉ sang lệnh tiếp theo

Thực hiện lệnh ADD X

t3: MAR ← MBR[addr] ← địa chỉ toán hạng X t4: MBR ← M[MAR] ← đọc giá trị tại X t5: ACC ← ACC + MBR ← ALU thực hiện phép cộng

Thực hiện lệnh LOAD X

─── Nhập lệnh (giống mọi lệnh) ─── C0: MAR ← PC C1: MBR ← M[MAR] C4: IR ← MBR[op] C2: PC ← PC + 1 ─── Thực hiện ─── C5: MAR ← MBR[addr] ← địa chỉ X C1: MBR ← M[MAR] ← giá trị tại X C7: ACC ← MBR ← nạp vào ACC

Thực hiện lệnh STORE X

─── Nhập lệnh (giống mọi lệnh) ─── C0: MAR ← PC C1: MBR ← M[MAR] C4: IR ← MBR[op] C2: PC ← PC + 1 ─── Thực hiện ─── C5: MAR ← MBR[addr] ← địa chỉ X C6: MBR ← ACC ← đưa ACC vào đệm C8: M[MAR] ← MBR ← ghi ra bộ nhớ

Ví dụ đầy đủ: ADD 1000

Trạng thái ban đầu: PC = 0100, ACC = 0101 (= 5), ô nhớ 1000 chứa 0111 (= 7).

═══ NHẬP LỆNH ═══ t1: MAR ← PC → MAR = 0100 t2: MBR ← M[0100] → MBR = 00111000 IR ← MBR[OP] → IR = 0011 (ADD) PC ← PC+1 → PC = 0101 ═══ THỰC HIỆN ═══ t3: MAR ← MBR[addr] → MAR = 1000 t4: MBR ← M[1000] → MBR = 0111 (=7) t5: ACC ← ACC + MBR → ACC = 0101+0111 = 1100 (=12) ✓

Checklist ôn thi — Bài 1

Lịch sử & Von Neumann

  • ☐ 4 thế hệ máy tính và công nghệ đặc trưng của từng thế hệ
  • ☐ Mốc thi: ENIAC 1943 · Von Neumann / EDVAC 1945 · Intel 4004 1971 · IBM PC 1981
  • ☐ 3 nguyên lý Von Neumann (lưu trữ chung · tuần tự · bus)
  • ☐ 3 thành phần chính + Bus hệ thống

Hệ nhị phân

  • ☐ Chuyển thập phân → nhị phân (phần nguyên): chia cho 2, đọc từ dưới lên
  • ☐ Chuyển thập phân → nhị phân (phần thập phân): nhân với 2, đọc từ trên xuống
  • ☐ Tính bù 2: đảo bit rồi cộng 1. Kiểm tra bằng phép cộng (tổng = 0)
  • ☐ Cấu trúc IEEE 754: S (1 bit) · E (8 bit) · F (23 bit) và công thức V = (−1)ˢ × 2^(E−127) × 1.F

CPU & Chu kỳ lệnh

  • ☐ Chức năng 8 thành phần CPU: PC, IR, MAR, MBR, ACC, PSW, CU, ALU
  • ☐ 6 bước chu kỳ lệnh theo thứ tự đúng
  • ☐ Vi thao tác chu kỳ nhập lệnh (t1, t2): MAR←PC, MBR←M[MAR], IR←MBR[op], PC←PC+1
  • ☐ Vi thao tác ADD X, LOAD X, STORE X — đặc biệt phần thực hiện
Mẹo ôn thi Phần nhập lệnh (t1–t2) giống nhau cho mọi lệnh. Chỉ cần nhớ thêm phần thực hiện riêng của từng lệnh: ADD cộng vào ACC, LOAD nạp vào ACC, STORE ghi ra bộ nhớ.
