Cơ sở dữ liệu

Thiết kế, xây dựng và truy vấn cơ sở dữ liệu quan hệ. Nền tảng không thể thiếu cho mọi ứng dụng.

Giới thiệu CSDL

Cơ sở dữ liệu (Database) là tập hợp dữ liệu có tổ chức, được lưu trữ và quản lý bởi Hệ quản trị CSDL (DBMS).

DBMS phổ biến MySQL · PostgreSQL · SQLite · Microsoft SQL Server · Oracle

Tại sao cần CSDL?

  • Lưu trữ dữ liệu lâu dài, có cấu trúc
  • Tránh trùng lặp, đảm bảo nhất quán
  • Nhiều người dùng cùng truy cập an toàn
  • Truy vấn nhanh với khối lượng dữ liệu lớn

Mô hình quan hệ

Mô hình quan hệ tổ chức dữ liệu thành các bảng (table), mỗi bảng gồm hàng (row/record) và cột (column/field).

Các khái niệm chính

Khái niệmÝ nghĩa
Primary Key (PK)Khóa chính — định danh duy nhất mỗi hàng
Foreign Key (FK)Khóa ngoại — liên kết với PK của bảng khác
Relationship1-1, 1-N, N-N giữa các bảng
IndexTăng tốc truy vấn trên cột thường dùng

SQL cơ bản

SQL (Structured Query Language) là ngôn ngữ chuẩn để thao tác với CSDL quan hệ.

DDL — Định nghĩa cấu trúc

CREATE TABLE SinhVien (
  MaSV     INT PRIMARY KEY AUTO_INCREMENT,
  HoTen    VARCHAR(100) NOT NULL,
  NgaySinh DATE,
  Diem     FLOAT DEFAULT 0
);

ALTER TABLE SinhVien ADD COLUMN Email VARCHAR(100);
DROP TABLE SinhVien;

DML — Thao tác dữ liệu

-- Thêm
INSERT INTO SinhVien (HoTen, Diem) VALUES ('Minh Đức', 8.5);

-- Đọc
SELECT HoTen, Diem FROM SinhVien WHERE Diem >= 8 ORDER BY Diem DESC;

-- Cập nhật
UPDATE SinhVien SET Diem = 9.0 WHERE MaSV = 1;

-- Xóa
DELETE FROM SinhVien WHERE MaSV = 1;

JOIN — Kết hợp bảng

SELECT sv.HoTen, mh.TenMH, kq.Diem
FROM SinhVien sv
INNER JOIN KetQua kq ON sv.MaSV = kq.MaSV
INNER JOIN MonHoc mh ON kq.MaMH = mh.MaMH;
Mẹo thi Nhớ thứ tự: SELECT → FROM → JOIN → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT

Chuẩn hóa CSDL

Chuẩn hóa (Normalization) giúp loại bỏ trùng lặp và phụ thuộc hàm không cần thiết trong thiết kế CSDL.

Dạng chuẩnYêu cầu
1NFMỗi ô chỉ chứa một giá trị nguyên tử
2NF1NF + không có phụ thuộc hàm một phần
3NF2NF + không có phụ thuộc hàm bắc cầu
BCNFDạng mạnh hơn 3NF
Lưu ý Chuẩn hóa quá mức có thể làm chậm truy vấn (nhiều JOIN). Trong thực tế, thường dừng ở 3NF hoặc BCNF.
