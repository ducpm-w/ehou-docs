Lập trình OOP

Lập trình hướng đối tượng — mô hình lập trình mạnh mẽ nhất, nền tảng của Java, C++, Python và hầu hết ngôn ngữ hiện đại.

OOP là gì?

OOP (Object-Oriented Programming) là mô hình lập trình tổ chức code xung quanh đối tượng (object) — kết hợp dữ liệu và hành vi vào một đơn vị.

Lợi ích Tái sử dụng code · Dễ bảo trì · Mô hình hóa thực tế · Giảm phức tạp

4 tính chất OOP

Encapsulation (Đóng gói)Ẩn chi tiết, chỉ expose interface cần thiết

Class & Object

Class là bản thiết kế (blueprint). Object là thực thể được tạo ra từ class đó.

// Java
public class SinhVien {
    // Thuộc tính (fields)
    private String hoTen;
    private float diem;

    // Constructor
    public SinhVien(String hoTen, float diem) {
        this.hoTen = hoTen;
        this.diem = diem;
    }

    // Phương thức (method)
    public void hienThi() {
        System.out.println(hoTen + " — Điểm: " + diem);
    }

    // Getter
    public String getHoTen() { return hoTen; }
}

// Tạo object
SinhVien sv = new SinhVien("Minh Đức", 8.5f);
sv.hienThi(); // Minh Đức — Điểm: 8.5
Access Modifier private: chỉ trong class · protected: trong class và subclass · public: mọi nơi

Kế thừa (Inheritance)

Class con (subclass) kế thừa toàn bộ thuộc tính và phương thức public/protected từ class cha (superclass).

public class NguoiDung {
    protected String ten;
    public NguoiDung(String ten) { this.ten = ten; }
    public void chaoHoi() { System.out.println("Xin chào, " + ten); }
}

// SinhVien kế thừa NguoiDung
public class SinhVien extends NguoiDung {
    private String maSV;

    public SinhVien(String ten, String maSV) {
        super(ten); // gọi constructor cha
        this.maSV = maSV;
    }

    @Override
    public void chaoHoi() { // Override — đa hình
        System.out.println("Sinh viên " + ten + " — MSSV: " + maSV);
    }
}
Mẹo Dùng @Override annotation để trình biên dịch kiểm tra bạn có override đúng phương thức không.
